The backend uses SQLite for development and production. SQLite does not run in a
separate process. It is a library which is dynamically linked to the server. The
database is stored on disk as database.db

For more information, see the Rusqlite readme:

    https://github.com/rusqlite/rusqlite#notes-on-building-rusqlite-and-libsqlite3-sys

To see exactly how SQLite is installed during deployment, see the Heroku build
script:

    https://github.com/willmcpherson2/notcord-buildpack/blob/master/bin/compile
